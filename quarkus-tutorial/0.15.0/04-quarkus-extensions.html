Quarkus Extensions

At the end of this chapter you will be able to :

  • What is Quarkus Extensions ?

  • How to add them to your Quarkus project?

What are extensions ?

Extensions are set of dependencies that can added to the Quarkus project in need of a specific features e.g. health checks. Extensions configure, boot and integrate a framework or technology into your Quarkus application. They also do all of the heavy lifting of providing the right information to GraalVM for your application to compile natively.

Listing Extensions

The Quarkus maven plugin can list all available supported extensions.

Navigate to the tutorial’s work folder:

cd $TUTORIAL_HOME/work

The instructions of this section assumes your project name to be my-quarkus-project

To list the available extensions, run the following command from your project directory.

cd my-quarkus-project && \
./mvnw quarkus:list-extensions

Health checks

If you had observed, the quarkus application deployed on Kubernetes does not have any health checks on it. As Kubernetes best practice its highly recommended that health checks be added to the all the deployments.

Health checks in Kubernetes can be added using Liveliness and Readiness probes.

Adding new extensions

We can add quarkus-smallrye-health extension to allow Quarkus application to be probed on the REST endpoint called health to know about its current health.

To add quarkus-smallrye-health to the Quarkus application, run the following command from my-quarkus-project project directory:

./mvnw quarkus:add-extension -Dextension="quarkus-smallrye-health"

After successful execution of the above command, you will have the following dependency added to the my-quarkus-project project’s pom.xml:

pom.xml
...
    <dependency>
      <groupId>io.quarkus</groupId>
      <artifactId>quarkus-smallrye-health</artifactId>
    </dependency>
...

Rebuild container images

Let us rebuild the application after adding the health check extensions

  • JVM mode

  • Native mode

./mvnw -DskipTests clean package

After successful maven build, run the following command to build te container:

docker build -f src/main/docker/Dockerfile.jvm -t example/my-quarkus-project-jvm .

./mvnw -DskipTests clean package -Pnative -Dnative-image.docker-build=true (1)

1 Using the -Dnative-image.docker-build=true is very important as need a linux native binary what will be containerized.
Native compilation will take few minutes to complete.

After successful maven build, run the following command to build te container:

docker build -f src/main/docker/Dockerfile.native -t example/my-quarkus-project-native .

Checking health check endpoint

Lets start the newly built container:

  • JVM mode

  • Native mode

docker run -it --rm -p 8080:8080 example/my-quarkus-project-jvm

docker run -it --rm -p 8080:8080 example/my-quarkus-project-native

After adding smallrye-health extension we should have REST endpoint called /health available. Lets invoke the /health endpoint to check the health of the Quarkus application.

Invoke Service

The simple health REST URI can be called via browser using http://localhost:8080/health or using CLI like:

curl localhost:8080/health

The REST call about should return an response like:

{
    "checks": [],
    "outcome": "UP"
}

Add probes to Kubernetes Quarkus Application

Before we add the probes to the Kubernetes application, we need to push the rebuilt containers to the container registry.

  • JVM mode

  • Native mode

docker tag example/my-quarkus-project-jvm $APP_IMAGE && \
docker push $APP_IMAGE

docker tag example/my-quarkus-project-native $APP_IMAGE && \
docker push $APP_IMAGE

Once the rebuilt containers has been successfully pushed to the container registry run the following command to add the probes to the kubernetes deployment.

  • kubectl

  • oc

cp $TUTORIAL_HOME/templates/probes.yaml $TUTORIAL_HOME/work/my-quarkus-project/probes.yaml

Apply the probes to the existing $APP_NAME Kubernetes deployment:

kubectl patch deployment $APP_NAME -p "$(cat $TUTORIAL_HOME/work/my-quarkus-project/probes.yaml)"

oc set probe deployment $APP_NAME --readiness=true --liveness=true --get-url=http://:8080/health

Invoke the Kubernetes Service

Adding probes will cause the redeployment of the Kubernetes application. Wait for the application to be in Running state before you can Invoke the Service. The status of the application can be checked with the command:

  • kubectl

  • oc

kubectl get pods -w

oc get pods -w
